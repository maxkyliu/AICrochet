## ADDED Requirements

### Requirement: SQLite dataset store with versioned schema
The system SHALL maintain a SQLite database at `/data/aicrochet.db` containing a `training_records` table and a `feedback_corrections` table. The schema version SHALL be stored in a `schema_version` table and migrations applied automatically on startup.

#### Scenario: Database created on first run
- **WHEN** the data pipeline runs for the first time and `/data/aicrochet.db` does not exist
- **THEN** the database is created with all tables and `schema_version` set to 1

#### Scenario: Migration applied on version mismatch
- **WHEN** the codebase schema version is higher than the stored version
- **THEN** all pending migrations are applied in order before any reads or writes

---

### Requirement: Training records include source provenance and quality score
Each `training_records` row SHALL store: `id`, `source_type`, `source_url`, `pattern_id`, `part_name`, `primitive_type`, `scale`, `diameter_profile` (JSON array), `stitch_counts` (JSON array), `terminology`, `quality_score` (0.0–1.0), `is_synthetic` (boolean), `created_at`.

#### Scenario: Scraped record stored
- **WHEN** the normalizer produces a valid part record from a Ravelry pattern
- **THEN** a row is inserted with `is_synthetic=false` and `quality_score` derived from normalizer confidence

#### Scenario: Synthetic record flagged
- **WHEN** a diameter profile is generated by scaling the hardcoded base profiles
- **THEN** the record is inserted with `is_synthetic=true` so it can be filtered or downweighted during training

---

### Requirement: Dataset export produces train/val split
The dataset tooling SHALL export a train/val split (default 80/20) as JSON or CSV, stratified by `primitive_type` so all types are represented proportionally in both splits. Records with `quality_score < 0.5` SHALL be excluded from exports by default.

#### Scenario: Stratified split produced
- **WHEN** `dataset export --split train` is called
- **THEN** 80% of each primitive type's records are included, and no record appears in both the train and val exports

#### Scenario: Low-quality records excluded by default
- **WHEN** exporting without explicit quality threshold override
- **THEN** records with quality_score < 0.5 are excluded from the export

---

### Requirement: Dataset statistics reported on demand
The dataset tooling SHALL report: total record count, count per primitive type, synthetic vs real ratio, mean quality score, date range of collected patterns, and count of feedback corrections incorporated.

#### Scenario: Stats command output
- **WHEN** `dataset stats` is run
- **THEN** a summary table is printed to stdout with all required metrics within 2 seconds
